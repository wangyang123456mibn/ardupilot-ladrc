1
烧录固件
▼

使用 GitHub Actions 编译生成的 ArduCopter LADRC 固件（.bin 或 .hex 文件）进行烧录。

烧录步骤

  1. 打开 STM32CubeProgrammer
  2. 通过 ST-Link 或 USB DFU 模式连接飞控
  3. 选择固件文件（.bin 或 .hex）
  4. 设置烧录起始地址为 0x08010000
  5. 点击 Download 烧录
  6. 烧录完成后断开 ST-Link，通过 USB 连接 Mission Planner
⚠

注意：ArduPilot 在 STM32 上的启动地址为 0x08010000（偏移 64KB，为 bootloader 留出空间）。请勿烧录到 0x08000000，否则会覆盖 bootloader 导致无法启动。

ℹ

如果使用 .apj 文件，可以直接在 Mission Planner 中通过 Initial Setup → Install Firmware → Load custom firmware 进行烧录。

2
遥控器校准
▼

在配置任何参数之前，先完成遥控器校准。

操作步骤

  1. 确保 ELRS 接收器已连接到飞控的 UART1 (SERIAL1) 端口
  2. 遥控器 (RadioMaster Pocket) 与 ELRS 接收器已完成对频绑定
  3. 打开 Mission Planner，连接飞控
  4. 进入 初始设置 → 必要硬件 → 遥控器校准
  5. 点击 校准遥控
  6. 按照提示移动所有摇杆和开关，确保所有通道（CH1-CH6）都有正确的行程范围
  7. 确认各通道方向正确：
    • CH1：横滚（Roll）— 左右移动
    • CH2：俯仰（Pitch）— 上下移动
    • CH3：油门（Throttle）— 上下移动
    • CH4：偏航（Yaw）— 左右移动
    • CH5：辅助通道 1
    • CH6：飞行模式切换（三段开关）
  8. 点击 完成时点击 保存校准数据
ℹ

ELRS 使用 CRSF 协议，通道值为 1000-2000μs（标准 PWM 范围）。校准时确保所有通道的最低值约为 1000，最高值约为 2000。

3
串口配置
▼

配置飞控的串口，使 ELRS 接收器和 MAVLink 遥测正常工作。

参数值用途
SERIAL1_PROTOCOL 23 ELRS CRSF 接收器协议
SERIAL1_BAUD 57 57600 baud（CRSF 协议标准波特率）
SERIAL2_PROTOCOL 2 MAVLink2 遥测输出（连接地面站）
ℹ

SERIAL1 连接 ELRS 接收器（RX/TX 交叉连接），SERIAL2 可连接数传模块或 USB 用于 MAVLink 遥测。

4
机型配置
▼

将飞控设置为 BiCopter（双旋翼）机型。ArduCopter 的 FRAME_CLASS=10 为 BiCopter 模式。

参数值含义
FRAME_CLASS 10 BiCopter（双旋翼飞行器）
FRAME_TYPE 1 X 型（BiCopter 模式下忽略此值，设为 1 即可）
AHRS_ORIENTATION 0 标准方向（飞控正面朝前安装）
ℹ

修改 FRAME_CLASS 后需要重启飞控才能生效。Mission Planner 会自动提示重启。

5
舵机输出配置（BiCopter 特有）
▼

BiCopter 模式需要 2 个电机输出 + 2 个倾斜舵机输出。飞控通过控制舵机角度来调整电机推力方向，从而实现俯仰和偏航控制。

参数值含义
SERVO1_FUNCTION 33 Motor1（左电机）
SERVO2_FUNCTION 34 Motor2（右电机）
SERVO3_FUNCTION 38 TiltServo1（左倾斜舵机）
SERVO4_FUNCTION 39 TiltServo2（右倾斜舵机）
⚠

V-tail 舵机说明：本机使用的 PTK 7455MG 舵机为 V-tail 配置（控制俯仰和偏航），并非标准 BiCopter 的倾斜舵机。但在 ArduCopter 的 FRAME_CLASS=10 BiCopter 模式中，TiltServo 的功能正是通过差动控制来同时影响俯仰和偏航，因此 V-tail 舵机的效果与此类似。

ℹ

如果标准 BiCopter 模式的飞行效果不满足需求（例如 V-tail 的混控方式与 TiltServo 预期不同），可以考虑改用 FRAME_CLASS=0（Quad）+ 自定义混控矩阵。但建议先用标准 BiCopter 模式进行测试。

舵机行程设置

根据 PTK 7455MG 舵机的实际行程调整以下参数（默认值通常即可）：

参数建议值含义
SERVO3_MIN 1000 TiltServo1 最小 PWM
SERVO3_MAX 2000 TiltServo1 最大 PWM
SERVO4_MIN 1000 TiltServo2 最小 PWM
SERVO4_MAX 2000 TiltServo2 最大 PWM
6
飞行模式配置
▼

配置三段开关控制的飞行模式切换。使用 CH6（三段开关）进行模式切换。

三段开关 PWM 映射说明

ArduCopter 将 CH6 的 PWM 范围均分为 6 个档位，三段开关对应如下：

开关位置PWM 范围对应档位 (Position)
低档 ~1000-1233μs Position 1
中档 ~1234-1766μs Position 4
高档 ~1767-2000μs Position 6
⚠

关键：三段开关的中间位置对应的是 Position 4（不是 Position 2！）。因此必须配置 FLTMODE4 而不是 FLTMODE2 来设置中档模式。

正确的飞行模式参数配置

参数值含义
FLTMODE_CH 6 使用 CH6 三段开关切换飞行模式
FLTMODE1 0 Position 1（低档）= Stabilize 自稳定模式
FLTMODE2 -1 Position 2 = 禁用
FLTMODE3 -1 Position 3 = 禁用
FLTMODE4 2 Position 4（中档）= AltHold 定高模式
FLTMODE5 -1 Position 5 = 禁用
FLTMODE6 5 Position 6（高档）= Loiter 定点模式
INITIAL_MODE 2 解锁时默认进入 AltHold 定高模式
ℹ

模式说明：
- Stabilize (0)：自稳定模式，手动控制油门和姿态角度。适合首次试飞。
- AltHold (2)：定高模式，自动保持高度，手动控制位置。需要气压计。
- Loiter (5)：定点模式，自动保持位置和高度。需要 GPS 和气压计。

传感器配置
▼

配置传感器使能状态。室内测试时需要禁用 GPS 和指南针。

参数值含义
GPS1_TYPE 1 uBlox GPS（室内测试时设为 0 禁用）
COMPASS_USE 1 使用指南针（室内测试时设为 0 禁用）
COMPASS_ENABLE 1 启用指南针（室内测试时设为 0 禁用）
ARMING_CHECK 0 禁用所有解锁前检查（测试阶段使用）
⚠

测试阶段安全警告：ARMING_CHECK=0 会禁用所有解锁前安全检查，包括 GPS 锁定、指南针校准、加速度计校准等。这仅适用于室内测试阶段。室外飞行前必须恢复安全检查，建议设为 ARMING_CHECK=1（启用所有默认检查）。

室内测试配置

在室内测试时，使用以下传感器配置：

参数值说明
GPS1_TYPE 0 禁用 GPS
COMPASS_USE 0 禁用指南针使用
COMPASS_ENABLE 0 禁用指南针
ARMING_CHECK 0 禁用所有检查
8
电调校准
▼

电调（ESC）需要校准油门行程范围，确保油门信号与电机转速正确对应。

方法 1：ArduCopter 原生自动校准（推荐）

  1. 断开电机电源（非常重要！ 拔掉电池，仅保留 USB 供电）
  2. 在 Mission Planner 中设置 ESC_CALIBRATION=3
  3. 写入参数后重启飞控
  4. 飞控重启后会自动发送最大油门信号（2000μs）
  5. 此时接通电机电源（连接电池）
  6. 等待电调发出确认音（通常为"哔-哔-"两声）
  7. 约 5 秒后飞控自动发送最小油门信号（1000μs）
  8. 电调再次发出确认音，校准完成
  9. 设置 ESC_CALIBRATION=0（关闭校准模式）
  10. 重启飞控
⚠

安全警告：电调校准时必须先断开电机电源！在发送最大油门信号（步骤 6）之前，确保螺旋桨已拆下或电机无法转动。校准完成后接通电源时，保持螺旋桨远离人体和障碍物。

方法 2：RCPassThru 手动校准

  1. 在 Mission Planner 中进入 初始设置 → 必要硬件 → 电机测试
  2. 选择 RCPassThru 模式
  3. 选择要校准的电机通道（例如 CH1、CH2）
  4. 使用遥控器油门摇杆手动发送最大/最小信号进行校准
  5. 每个电调重复此过程
ℹ

如果电调支持 BLHeli_S 或 BLHeli_32，也可以使用 BLHeliSuite 通过 USB 编程器直接校准和配置电调参数。

9
PID 基础调参
▼

配置姿态控制器的基础参数。由于 LADRC 已内置到固件中，PID 参数只需设置基础值，LADRC 的线性自抗扰控制会自动补偿扰动。

姿态控制参数

参数值含义
ATC_ANGLE_PITCH_MAX 45 最大俯仰角限制（度）
ATC_ANGLE_ROLL_MAX 45 最大横滚角限制（度）
ATC_RATE_PITCH 10 俯仰角速度控制增益
ATC_RATE_ROLL 10 横滚角速度控制增益
ATC_RATE_YAW 5 偏航角速度控制增益
ATC_THR_MIX_MAN 0.5 手动油门混合比例（0-1）

电机输出参数

参数值含义
MOT_SPIN_MIN 0.1 最低怠速（10% 油门），防止电机停转
MOT_SPIN_MAX 0.95 最大转速限制（95%），保留安全余量
✓

LADRC 优势：传统 PID 调参需要反复调整 P、I、D 三个参数以平衡响应速度和稳定性。LADRC 通过线性扩张状态观测器（LESO）实时估计并补偿总扰动，因此 PID 参数只需设置合理的初始值，系统会自动处理大部分动态变化。

⚠

BiCopter 特殊注意：BiCopter 仅有 2 个电机，悬停稳定性比四旋翼差。建议初始测试时： - 降低 ATC_ANGLE_PITCH_MAX 和 ATC_ANGLE_ROLL_MAX 到 30 度 - 适当降低 ATC_RATE_PITCH 和 ATC_RATE_ROLL 到 8 - 测试稳定后再逐步增大

10
LADRC 说明
▼

LADRC（Linear Active Disturbance Rejection Control，线性自抗扰控制）已编译进固件中，默认启用。以下是 LADRC 的核心参数和说明。

LADRC 核心参数（编译时配置）

宏定义默认值说明
LADRC_ENABLED 1 启用 LADRC（设为 0 退化为纯 PID）
LADRC_WO 10.0 观测器带宽 (rad/s)，值越大观测器响应越快但噪声越敏感
LADRC_B0 1.0 控制通道增益，对应系统的控制输入系数
LADRC_ORDER 1 LESO 阶数（1=一阶 LESO，2=二阶 LESO）

工作原理

LADRC 在每个 PID 控制器内部运行一个线性扩张状态观测器（LESO），实时估计系统状态和总扰动：

  • z1：被控量的观测值（角度或角速度）
  • z2：总扰动（包括未建模动态、外部干扰、参数变化等）的估计值
  • z3：二阶 LESO 时的扰动变化率估计（一阶模式下未使用）

观测到的总扰动会被前馈补偿，大幅降低 PID 控制器需要处理的残余误差。

日志查看 LADRC 补偿量

在 Mission Planner 中可以查看 LADRC 的实时补偿量：

  1. 进入 数据分析 / DataFlash Logs
  2. 下载飞行日志（.bin 文件）
  3. 在日志中查找 PIDx.LADRC 字段（x 为 PID 实例编号）
  4. LADRC 值为观测器估计的扰动补偿量，正值表示正向补偿，负值表示反向补偿
PID 日志字段说明： P - 比例项输出 I - 积分项输出 D - 微分项输出 DFF - 前馈微分项输出 LADRC - LADRC LESO 扰动补偿量（本固件新增）
ℹ

调整建议：
- LADRC_WO：如果飞行中出现高频振荡，尝试降低到 5.0-8.0；如果响应迟钝可增大到 12.0-15.0
- LADRC_B0：通常保持 1.0，如果已知系统的控制增益特性可以微调
- LADRC_ORDER：一阶 LESO 适用于大多数姿态控制场景；二阶 LESO 可以跟踪更快变化的扰动但计算量更大
- 修改以上参数需要重新编译固件
